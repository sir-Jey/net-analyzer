Пассивный анализатор сетевого трафика.

Что умеет

- Перехватывает TCP, UDP, ICMP пакеты
- Для TCP: определяет операционную систему по TTL, Window Size и опциям (MSS, SACK, Timestamp, Window Scale), вычисляет расстояние в хопах, показывает DF и IP ID
- Для UDP и ICMP: выводит базовую информацию (адреса, порты, длину, тип ICMP-сообщения)
- Фильтрует трафик по протоколу и порту (через BPF)
- Завершается после заданного количества SYN-пакетов или по Ctrl+C с выводом статистики

Зависимости

- libpcap (dev)
- gcc (или clang)

Установка

MacOS

```bash
brew install libpcap
```

### Linux (Ubuntu/Debian)

```bash
sudo apt-get install libpcap-dev
```

## Компиляция и запуск

```bash
gcc -o net-analyzer main.c -lpcap
sudo ./net-analyzer
```

## Опции

| Опция | Действие |
|-------|----------|
| `-i <интерфейс>` | Интерфейс для прослушки (по умолчанию: выбирается первый) |
| `-c <количество>` | Завершиться после N SYN-пакетов (по умолчанию: бесконечно) |
| `-p <tcp/udp/icmp>` | Фильтр по протоколу (по умолчанию: tcp) |
| `--port <номер>` | Фильтр по порту (для tcp и udp) |
| `-h` | Справка |

## Примеры

```bash
# Запуск с фильтром по протоколу и порту
sudo ./net-analyzer -p udp --port 53
sudo ./net-analyzer -p icmp
```

## Пример вывода

```
192.168.0.4:54321 -> 93.184.216.34:80 | расстояние=7 роутеров | df=1 id=54321 | предполагаемая ОС=Linux | win=65535 | mss=1460 sack=yes wscale=8 tsval=12345678 tsecr=0
```

## Остановка

Ctrl+C — завершение с выводом статистики.
